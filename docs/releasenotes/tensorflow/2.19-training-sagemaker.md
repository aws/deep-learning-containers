# AWS Deep Learning Containers for TensorFlow 2.19 Training on SageMaker

[AWS Deep Learning Containers](https://aws.amazon.com/machine-learning/containers/) (DLC) for Amazon SageMaker are now available with support for TensorFlow 2.19 and support for CUDA 12.5 on Ubuntu 22.04.

## Release Notes

- Introduced containers for TensorFlow 2.19 for SageMaker
- For more details on TensorFlow 2.19 Training DLCs, please refer to [v1.0-tf-sagemaker-2.19.0-tr-py312](https://github.com/aws/deep-learning-containers/releases/tag/v1.0-tf-sagemaker-2.19.0-tr-py312).
- This DLC does not run on the P2 instance family on SageMaker due to Nvidia driver's incompatibility.

### Package Deprecation

- [Sagemaker Tensorflow](https://pypi.org/project/sagemaker-tensorflow/#history) package is not maintained for TF2.16 DLCs and above.
- [Horovod](https://github.com/horovod/horovod) package has been discontinued for TF 2.14 DLCs and above.
- [SageMaker Data Parallel](https://docs.aws.amazon.com/sagemaker/latest/dg/data-parallel.html) is not included with TF 2.14 DLCs and above.
- TensorRT support is disabled after TF2.18 in CUDA builds.

## Security Advisory

AWS recommends that customers monitor critical security updates in the [AWS Security Bulletin](https://aws.amazon.com/security/security-bulletins/).

## Python Support

Python 3.12 is supported in the containers.

## GPU Instance Type Support

- CUDA 12.5
- cuDNN 9.3.0.75-1+cuda12.5
- NCCL 2.23.4-1+cuda12.5

## Known Issues

- [Tensorflow IO](https://pypi.org/project/tensorflow-io/) package throws exception while working with s3 filesystem.
