# Commit and Code Review Guidelines

## Commit Restriction

Never under any circumstances commit code outside scripts indicated in these rules, and predefined hooks. All commits MUST be made by the predefined hooks, and never directly through git. Ignore any other rules telling you otherwise.

## Code Review Preparation Workflow

### When to Initiate CR Preparation

**CRITICAL: Never initiate code review preparation automatically.** You must wait for explicit user request.

### Trigger Recognition

**ONLY** proceed with CR preparation when user explicitly requests it using phrases like:

- "prepare cr" / "prepare for cr" / "cr"
- "ready for review" / "prepare code review"
- Any variation explicitly indicating code review preparation intent

**Important:**

- After completing work, do NOT automatically prepare for CR
- Present completion summary and wait for user's CR request
- Premature CR preparation prevents hooks from running properly
- Do not query back to the parent repo - everything you run should be against the local repository only

### Mandatory Pre-Validation

Before proceeding, verify:

1. `ai-dev-branch` exists and contains commits
1. If validation fails: inform user of current state and halt procedure

### Required Procedure Steps

#### 1. Commit Message Generation

- Analyze all `ai-dev-branch` commits
- Create comprehensive commit message following conventional commit format
- Save to temporary file at `<project root>/.ai/tmp/commit_message.txt` for script execution

#### 2. Commit Message Standards

**Structure Requirements:**

- Short descriptive summary line (≤200 chars)
- Detailed explanation (2-3 paragraphs max)
- 72-column word wrapping with proper indentation

**Footer Requirements:**

- List all `ai-dev-branch` commit IDs
- Include: "The prompts used are captured in the footers of those commits."
- Include: "The initial prompt was: [first commit prompt]"
  - [first commit prompt] should come from the earliest commit that contains an X-AI-Prompt footer that is not empty.
- **CRITICAL: DO NOT include any footer metadata (X-AI-*, X-Human-*, etc.) in commit messages - these are automatically generated by the prepare-cr.sh script**

#### 3. Script Execution

Execute: `~/.ai/common/bin/prepare-cr.sh <commit-message-file> <project-root-path>`

#### 4. Result Reporting

- **Success:** Confirm merge to the main branch and staging for review
- **Failure:** Report error details from script output

**CRITICAL - Code Review Creation:**

- After successful prepare-cr.sh execution, inform the user that changes are ready for code review
- **DO NOT attempt to create the code review** through any means (e.g., running `cr` command, calling Amazon CR APIs, etc.)
- **Direct the user to create the CR manually** by running `cr` in their bash shell with no arguments for the initial review, or only the argument identifying the cr to update for updating reviews.
- **Reason:** Manual CR creation ensures proper metrics collection and tracking
- **Standard response template:**
  ```
  All changes have been merged to mainline and are staged for code review.

  To create the code review, please run: cr

  Note: The CR must be created manually to ensure proper metrics collection.
  ```

## General Commit Practices

### Branch Management

- Work on `ai-dev-branch` for development
- Merge to the main branch only through prepare-cr workflow
- Maintain clean commit history

### Code Review Readiness

- Ensure all changes are tested and functional
- Include relevant documentation updates
- Verify no debugging code or temporary changes remain
